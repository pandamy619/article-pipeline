# Пайплайн «поиск статей → публикация в Telegram»

Стек и архитектура под условия: **RTX 4080 (16 ГБ), русский язык, проверка человеком перед публикацией.** Источники: RSS, arXiv/Habr/Reddit, веб-поиск, другие Telegram-каналы.

---

## 1. Локальная нейросеть и железо

**Убьёт ли это RTX 4080?** Нет. Инференс LLM — штатная нагрузка для видеокарты, по стрессу обычно мягче тяжёлых игр (упор в пропускную способность памяти, а не постоянные 100% по чипу). Карта рассчитана на годы такой работы.

Что стоит сделать для долговечности и комфорта:

- Хорошая продувка корпуса, периодическая чистка от пыли, ядро в идеале < 80 °C.
- Power limit ~80–85% или лёгкий андервольт: −2–4% скорости, заметно меньше нагрев/шум/потребление.
- Реальное ограничение — не износ, а **16 ГБ VRAM**: они определяют максимальный размер модели.

**Рекомендуемая модель: Qwen3-14B** (квантизация Q4_K_M, ~12 ГБ VRAM, ~60 ток/с на 4080). Лучший баланс «качество русского / скорость / влезает с запасом под контекст». Рассуждения и ответы держит на русском без вставок на других языках.

Альтернативы:

- **Gemma 3 12B** — быстрее, но русский слабее.
- **Mistral Small 24B (Q4)** — умнее, но ~13 ГБ почти впритык, мало места под контекст.
- **Русскоязычные дообучения** (Vikhr, Saiga, T-pro/T-lite) — если захочется максимально «нативный» русский стиль; можно подключить как вторую модель под рерайт.

**Рантайм: Ollama** — для одного пользователя и пары запросов в минуту это идеально (минимум возни, один бинарь, REST API). vLLM нужен только при высокой нагрузке/параллелизме — для модерации с участием человека он избыточен.

**Эмбеддинги (для дедупа и фильтра): bge-m3** или multilingual-e5 — локально, мультиязычные, отлично различают близкие по смыслу статьи.

## 2. Архитектура по этапам

### Этап 0. Источники (ingest)

| Источник | Как забирать |
|---|---|
| RSS избранных сайтов | `feedparser` (Python) по списку URL |
| Habr | RSS-ленты хабов/потоков |
| arXiv | официальный arXiv API (по категориям/запросам) |
| Reddit | официальный API (PRAW) по сабреддитам |
| Веб-поиск | self-hosted **SearXNG** (бесплатно, без ключей) или платный Search API (Tavily/Brave/Serper) |
| Telegram-каналы | **Telethon** (user-сессия) для чтения чужих каналов |

> Важно: для чтения чужих каналов нужен user-аккаунт (Telethon), а не бот — боты не видят историю каналов, где не админ. Для **публикации** в свою группу — наоборот, обычный бот (Bot API).

### Этап 1. Сборщик

Нормализует всё в единый формат `{title, url, text, source, published_at}`, чистит HTML (`trafilatura`/`readability`), **дедуплицирует**: сначала по URL/хешу, потом по косинусной близости эмбеддингов (отсекает «ту же новость другими словами»).

### Этап 2. AI-фильтр релевантности

Локальная LLM (или эмбеддинги + LLM) оценивает каждую статью по твоим критериям и ставит оценку 0–10 + причину. Ниже порога — отбрасывается. Промпт описывает тематику канала, что «заходит», что нет. Дёшево и полностью локально.

### Этап 3. AI-рерайт на русском

Qwen3-14B делает из статьи готовый пост: краткое саммари/пересказ, нужный стиль и длину, заголовок, хэштеги, ссылку на источник, форматирование под Telegram (HTML/MarkdownV2, лимит 4096 символов). Для перевода англоязычных — тот же шаг.

### Этап 4. Модерация (человек) — ключевой шаг

Сгенерированный пост попадает в очередь на проверку. Самый простой и удобный вариант — **бот присылает черновик тебе в личку с кнопками** ✅ Опубликовать / ✏️ Править / ❌ Отклонить (inline-кнопки, `aiogram`). Альтернатива — простая веб-админка (FastAPI + минимальный фронт) со списком черновиков.

### Этап 5. Публикация

После аппрува бот постит в группу через Bot API (`sendMessage`/`sendPhoto`), сохраняет статус и `message_id` в БД, помечает источник как обработанный.

## 3. Хранилище и оркестрация

- **БД:** PostgreSQL (или SQLite для старта) — статьи, статусы (`new → filtered → drafted → pending → published/rejected`), история для дедупа.
- **Очередь/кэш:** Redis — задачи между этапами, рейт-лимиты.
- **Оркестрация (что чем запускать по расписанию):**
  - **n8n** (low-code, визуально) — если хочется собирать и менять флоу мышкой.
  - **Prefect / APScheduler** — если пишешь на Python и хочешь код.
  - **cron + скрипты** — самый простой старт.

## 4. Итоговый стек (рекомендация)

| Слой | Технология |
|---|---|
| Язык | Python 3.11+ |
| Локальный LLM | Ollama + **Qwen3-14B (Q4_K_M)** |
| Эмбеддинги | bge-m3 (через Ollama / sentence-transformers) |
| Сбор | feedparser, PRAW, arXiv API, Telethon, SearXNG |
| Чистка текста | trafilatura / readability-lxml |
| Telegram-бот | **aiogram 3** |
| БД / кэш | PostgreSQL + Redis (или SQLite на старте) |
| Оркестрация | n8n **или** Prefect/cron |
| Деплой | Docker Compose на твоей машине |

## 5. Поэтапный план (от простого к полному)

**MVP за вечер:** 1 RSS-источник → фильтр по ключевым словам → рерайт Qwen3 → бот шлёт черновик с кнопками в личку → публикация в группу. Без БД (JSON-файл), без Redis.

**V2:** добавить arXiv/Habr/Reddit + SearXNG, эмбеддинг-дедуп, PostgreSQL, оценку релевантности через LLM.

**V3:** мониторинг TG-каналов (Telethon), оркестрация (n8n/Prefect), картинки в постах, логи/метрики, расписание.

## 6. На что заранее обратить внимание

- **Лимиты Telegram:** ~20 сообщений/мин в группу, 4096 символов на сообщение, длинный текст → разбивать или слать как файл/несколько постов.
- **Авторские права:** публиковать полные тексты чужих статей рискованно — безопаснее саммари + ссылка на оригинал. Для TG-каналов уважать правила репоста.
- **Качество фильтра:** первые дни держать порог релевантности повыше и смотреть, что бот предлагает, потом калибровать промпт.
- **Стоимость:** при self-hosted (Ollama + SearXNG) внешних платежей нет; платные Search API нужны, только если SearXNG не устроит по качеству.

## 7. Деплой на Windows (Docker)

На Windows всё крутится поверх **WSL2** — Docker Desktop использует его как бэкенд. GPU (RTX 4080) пробрасывается в контейнеры, поэтому Ollama можно держать прямо в Docker, и она будет использовать видеокарту. То есть «всё в докере, насколько возможно» — реально.

**Поставить на Windows один раз:**

1. Свежий драйвер NVIDIA (с поддержкой CUDA — для Ada/4080 любой актуальный подойдёт).
2. WSL2 + Ubuntu: `wsl --install`.
3. Docker Desktop с включённым WSL2-бэкендом и интеграцией с Ubuntu.

**Проверка, что GPU виден в контейнере:**

```
docker run --rm --gpus all nvidia/cuda:12.4.0-base-ubuntu22.04 nvidia-smi
```

Видишь свою 4080 — проброс работает. Если `--gpus all` ругается на отсутствие nvidia runtime, внутри WSL2-дистрибутива:

```
sudo apt install -y nvidia-container-toolkit
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker
```

(С Docker Desktop обычно достаточно драйвера Windows; команды выше — для ручного Docker Engine в WSL2.)

**Что в контейнерах, а что на хосте:**

- **В Docker:** ollama (с GPU), postgres, redis, searxng, app (сборщик + фильтр + рерайт + бот aiogram), опционально n8n.
- **На хосте — только Docker Desktop.** Запасной вариант: если проброс GPU вдруг капризничает, поставить Ollama нативным Windows-приложением, а контейнеры обращаются к ней по `host.docker.internal:11434`; всё остальное остаётся в Docker.

**Нюансы Windows:**

- **Автозапуск:** включи в Docker Desktop «Start on login» + `restart: unless-stopped` у сервисов — пайплайн поднимется сам после перезагрузки.
- **Модели Ollama** держи в именованном volume (`ollama:/root/.ollama`), иначе 14B будет качаться заново.
- **Telethon-сессия:** для чтения чужих TG-каналов нужен user-логин. Первый вход — интерактивно, один раз сгенерировать `.session`-файл (например `docker compose run --rm app python make_session.py`), потом он лежит в `./sessions` и монтируется в контейнер. Боту (aiogram) сессия не нужна.
- **RAM для WSL2:** в `C:\Users\<ты>\.wslconfig` ограничь память (например 16–24 ГБ), иначе WSL2 съест всё.
- **Переносы строк:** для смонтированных скриптов держи LF, не CRLF, иначе bash в контейнере споткнётся.

Стартовые файлы рядом с документом: **`docker-compose.yml`** и **`.env.example`** (скопировать в `.env` и заполнить).
